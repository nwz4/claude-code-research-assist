# 登录态学术站点 fallback 规则

适用站点示例：
- IEEE Xplore
- ACM Digital Library
- Springer
- ScienceDirect

## 何时启用 Playwright

当出现以下任一情况时启用：
- WebFetch 只能拿到登录提示或空壳页面
- 用户明确说自己有账号
- 需要摘要、关键词、作者、DOI、下载入口等，而公开页面抓不到

## 标准流程

1. 打开浏览器会话
2. 进入目标论文落地页
3. 若用户尚未登录，让用户在浏览器内完成登录
4. 登录后优先抓取：
   - 标题
   - 作者
   - 摘要
   - 关键词
   - DOI
   - venue / year / publisher
   - 引用信息或下载入口
5. 只有在有明确需求时，再继续尝试 PDF 页面

## 原则

- 不凭空猜 PDF 链接
- 不绕过登录
- 不声称读取了无法访问的全文
- 需要引用时，说明该信息来自登录态页面

## 推荐最小读取目标

对科研调研来说，通常只要拿到这些就足够：
- 标题
- 作者
- 摘要
- DOI
- venue / year
- PDF 是否可下载

若这些信息齐全，通常已经足够进入研究报告。
