# 学术研究报告模板

## 1. 研究结论摘要
- 3–6 条最重要发现
- 区分：事实 / 推断 / 建议
- 若是核验任务，标记 supported / partial / unverified

## 2. 检索问题与范围
- 检索主题
- 关键词 / 同义词
- 年份范围
- 是否包含综述、基准、项目页、代码仓库
- 是否使用了登录态来源（如 IEEE）
- 本次模式：paper-search / research-think / paper-compare / claim-verify

## 3. 检索方法
- 使用了哪些来源
- 搜索通道与 Grok 通道分别做了什么
- 如何做去重
- 如何判定优先级
- 哪些来源可能仍不完整

## 4. 核心论文 / 核心来源清单
对每个条目建议列：
- 标题
- 作者
- 年份 / venue
- 一句话贡献总结
- 为什么重要
- 主来源 URL
- 备用来源 URL
- 代码 / 项目页

## 5. 关键 claims 与证据
对每条 claim 建议列：
- claim
- confidence
- support（supported / partial / unverified）
- evidence URLs
- note

## 6. 方法脉络与主题聚类
可按以下角度分组：
- 问题设定
- 方法路线
- 模态 / 任务类型
- 数据集 / 评测方式
- 与用户目标的相关性

## 7. 分歧点与对比结论
适用于 compare / verify：
- 哪些结论一致
- 哪些地方存在冲突
- 分歧来自方法、数据、设定还是来源质量

## 8. 值得优先精读的论文
给 3–8 篇，并说明：
- 为什么优先读
- 先读哪篇综述
- 再读哪篇代表作
- 哪篇适合看实现

## 9. 风险与未知项
- 还有哪些论文可能漏掉
- 哪些结论只基于摘要而非全文
- 哪些条目版本关系还需确认
- 哪些结果来自登录态页面
- 哪些 claim 还没有被搜索通道充分支持

## 10. Next Queries
- 下一轮建议检索词
- 值得继续验证的问题
- 需要用 Playwright 补抓的页面

## 11. Sources
- 按论文 / claim 对象列主来源和备用来源
- 对同一论文尽量只列一次主条目，再附 alternate sources
